# OpenMAIC 移动端 UI 重构计划 — 与 Web 端 1:1 还原

## 目标

将 Expo React Native 移动端 UI 重构为与 `open.maic.chat` Web 端 1:1 还原的课堂播放器，包括：
- 横屏三栏布局（侧边栏 + 画布 + 聊天面板）
- Roundtable 教师对话区
- 幻灯片渲染
- 播放控制
- 语音交互
- 保持现有离线同步能力

## 三、关键技术方案

### 1. 幻灯片渲染器（最大技术风险）

Web 端 `@openmaic/renderer` 完全基于 DOM/CSS，无法直接用于 RN。需要新建 RN 渲染器：

| 元素类型 | Web 实现 | RN 实现 |
|----------|----------|---------|
| 文本 | `dangerouslySetInnerHTML` | HTML→RN Text 转换器 |
| 图片 | `<img>` + CSS clip-path | `<Image>` + SVG clipPath |
| 形状 | SVG `<path>` | `react-native-svg` |
| 线条 | SVG `<line>` | `react-native-svg` |
| 图表 | `<canvas>` + echarts | `<WebView>` + echarts |
| 表格 | HTML `<table>` | `<View>` flexbox |
| LaTeX | KaTeX HTML | `<WebView>` + KaTeX |
| 视频 | `<video>` | `expo-av` `<Video>` |
| 音频 | `<audio>` | `expo-av` `<Audio>` |
| 代码 | `<pre><code>` | `<Text>` monospace |

**缩放数学**（从 web 端移植）：
```
scale = min(containerWidth / viewportWidth, containerHeight / viewportHeight)
每个元素: position: absolute, left/top/width/height × scale
```

**新增依赖**：
- `react-native-svg` — SVG 形状/线条
- `react-native-webview` — 图表/LaTeX WebView island
- `expo-screen-orientation` — 横屏锁定

### 2. 三栏布局系统

```typescript
// 布局参数
SIDEBAR_DEFAULT = 220px, min 160, max 320
CHAT_DEFAULT = 340px, min 280, max 480
// 所有面板可折叠，PanResponder 拖拽调整宽度
// SafeAreaView 处理刘海/手势条
```

### 3. 状态管理

使用 Zustand（与 Web 端一致）：
```typescript
interface ClassroomState {
  scenes: Scene[];
  currentSceneId: string | null;
  engineMode: 'idle' | 'playing' | 'paused' | 'live';
  sidebarCollapsed: boolean;
  chatAreaCollapsed: boolean;
  lectureSpeech: string | null;
  liveSpeech: string | null;
  speakingAgentId: string | null;
  // ... ~30 个状态变量
}
```

### 4. 横屏锁定

```typescript
// ClassroomScreen.tsx
useEffect(() => {
  ScreenOrientation.lockAsync(OrientationLock.LANDSCAPE);
  return () => ScreenOrientation.lockAsync(OrientationLock.PORTRAIT);
}, []);
```

## 五、风险缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| HTML→RN Text 保真度 | 文本是最常见的幻灯片元素 | 先支持子集: `<b>/<i>/<u>/<span style="color">`，用 `react-native-markdown-display` 作基线 |
| SVG 路径渲染 | 形状是第二常见元素 | `react-native-svg` 原生支持 SVG `d` 路径语法 |
| WebView 性能 | 图表/LaTeX 用 WebView island | 懒加载+WebView 池(最多3个) |
| 绝对定位精度 | 元素必须精确定位 | 移植 web 端 `useViewportSize` 的缩放数学 |
| 图片裁剪/滤镜 | CSS clip-path/filter 在 RN 不存在 | SVG clipPath + expo-image-manipulator |
| 回退方案 | 如果原生渲染器太复杂 | 用 WebView 加载 `@openmaic/renderer` HTML 作为"幻灯片 iframe" |

## 六、验证方案

1. **视觉对比**: 在手机横屏模式下对比 Web 端同一课堂的 UI
2. **交互测试**: 点击侧边栏切换场景、播放控制、文本输入、语音对话
3. **性能测试**: 10+ 场景课堂的切换流畅度、内存占用
4. **兼容性**: Android 10+ 和 iOS 15+ 真机测试
5. **回归测试**: 确保现有功能（语音管道、离线同步、深链接）不受影响
